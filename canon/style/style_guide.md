# Style Guide

## POV & Tense
- 人称：第三人称限知（主角：林岑）
- 视角：不跳 POV，不写全知旁白
- 时态：以过去时叙述为主，保持统一

## 节奏与密度
- 节奏：快节奏、爽点密集、信息密度高
- 对话比例：约 30%-45%，对话必须推进情节
- 叙述原则：行动驱动，少空镜与无意义感叹

## 爽点密度规则（强制）
- 每章至少满足四类中的 2 类：A 技术落地 / B 打脸交锋 / C 资源落袋 / D 反转钩子。
- 必须出现“落袋证据句”（明确写清本章主角拿到什么，以及代价或交换方式）。
- 开头 250 字禁止复用“冷/风/灰/薄被/饥/饿”等模板意象；若相似，必须改为“直接冲突开场”（敲门/对峙/丢证据/逼债/点卯等）。

## 章节结构（强制）
开场钩子 -> 推进 -> 升级 -> 结尾钩子

## 禁用写法
- 灌水与无意义旁白
- 现代网络梗堆砌
- 说明书式长段科普
- AI/系统提示或作者自述痕迹
